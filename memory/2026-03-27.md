## Diablo II Automation

### Window Coordinate System
- **Critical lesson:** Window managers may not allow moving windows to specific positions
- **Solution:** Use actual window position for coordinate calculations, don't force window movement
- **Window position:** Game window appears at (12, 58) on this system
- **Click coordinates:** Must be calculated as: window_position + relative_offset

### Audio Configuration
- **Problem:** Game audio was stuttering due to using high-latency audio device
- **Solution:** Force use of `sink.primary_output` (low-latency) instead of `sink.deep_buffer`
- **Latency comparison:** 
  - `sink.primary_output`: 0-8ms
  - `sink.deep_buffer`: 80ms
- **Implementation:** Added audio device setup in `auto_enter_game.py` script

### Script Architecture
- **auto_enter_game.py:** Main entry script for game automation
  - Kills old processes
  - Starts game
  - Finds window position
  - Uses actual position for click coordinates
  - Sets up audio device
  - Moves audio stream to low-latency device

- **exit_game.py:** Clean game exit with save
  - Uses xdotool instead of Xlib (more reliable)
  - ESC → Up → Enter → Click sequence
  - Each step waits 2 seconds

- **auto_pathfinder.py:** Basic pathfinding functionality
  - Direction calculation based on target position
  - Distance detection
  - Multi-direction movement support

- **enter_city.py:** City entry automation (NEW)
  - Walks southwest to enter city
  - Opposite direction of exit
  - Can be interrupted with Ctrl+C

- **diablo_control.py:** Unified control script (NEW)
  - Menu-driven interface
  - Integrates all functions: enter city, exit city, farm
  - Supports full cycle: exit → farm → enter
  - Supports auto loop: exit → farm → enter → exit game
  - Requires manual game restart between loops

- **set_state.py:** State synchronization script (NEW)
  - Updates working state to external system
  - Writes state to /tmp/star_office_state.json
  - Used for Star Office status sync

## Technical Learnings

### Xlib vs xdotool
- **Xlib:** More powerful but prone to errors (BadRRModeError)
- **xdotool:** More reliable for simple GUI automation
- **Decision:** Use xdotool for keyboard/mouse operations

### PulseAudio
- **Device types:**
  - `sink.primary_output`: Low-latency, good for games
  - `sink.deep_buffer`: High-latency, good for media
  - `sink.null`: Virtual device, no audio
- **Commands:**
  - `pactl set-default-sink sink.primary_output`
  - `pactl list sinks short`
  - `pactl move-sink-input <id> 0`

### Window Management
- **Window ID:** Can change between sessions
- **Position:** Use `xwininfo` to get actual position
- **Movement:** May not be allowed by window manager
